# Hardware and e-waste scenography

The hardware body should avoid the fantasy that computation is clean, immaterial, and sealed.

## Principle

```txt
The apparatus should show both living computation and electronic remains.
```

The work may combine a functioning controller with dead or obsolete components: racks, switches, patch panels, hard drives, memory sticks, power supplies, cables, labels, and discarded electronics.

## Living and dead hardware

- **Living:** controller, printer interface, network, storage, logging, generation process.
- **Dead:** recovered e-waste, obsolete enterprise hardware, failed components, inert infrastructure.
- **Accumulating:** printed paper, ink, dust, heat, operational logs.

The tension between these layers matters. The living machine is surrounded by the material afterlife of previous machines.

## Sourcing ethics

Hardware remains should be sourced through legal and responsible channels. If recovered from recycling cooperatives or e-waste partners, the project should not interrupt responsible disposal chains without clear agreement.

The installation should avoid fetishizing waste while ignoring labor, toxicity, and recycling infrastructure.

## Visual posture

The hardware should not become generic cyberpunk decoration. It should remain legible as infrastructure: labeled, cabled, heavy, specific, and materially accountable.

## Open questions

- How visible should the functioning controller be?
- Should dead hardware surround, support, or frame the printer?
- Should components be labeled with provenance?
- Should the installation reveal cloud/API dependency if the system uses remote computation?
